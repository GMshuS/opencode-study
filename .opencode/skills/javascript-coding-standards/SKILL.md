# 核心规范摘要
1. 基本风格：2空格缩进、字符串使用单引号、始终使用分号
2. 模块规范：使用 `const` 声明、`require()` 导入、`module.exports` 导出
3. 错误处理：使用 try/catch 或 Error-first 回调
4. 异步操作优先使用 async/await
5. 遵循标准目录结构（src/test/config/docs/public）
6. 安全实践：验证用户输入、避免 eval、使用 HTTPS
