# Go 代码审查检查清单

## 架构设计审查

- [ ] **模块化设计**
  - 包职责是否单一
  - 目录结构是否清晰
  - 依赖关系是否合理

- [ ] **初始化机制**
  - `init()` 函数使用是否恰当
  - 全局变量初始化是否安全
  - 资源清理是否有 `defer` 或显式清理函数

- [ ] **接口设计**
  - 接口定义是否简洁（`Interface Segregation`）
  - 是否遵循"接受接口，返回结构体"原则

## 并发安全审查

- [ ] **Goroutine 管理**
  - goroutine 是否有退出机制
  - 是否使用 `context` 控制生命周期
  - 是否存在 goroutine 泄漏风险

- [ ] **Channel 使用**
  - channel 是否有缓冲区溢出风险
  - channel 关闭是否正确
  - 是否存在 channel 泄漏（只写不读/只读不写）

- [ ] **数据竞争**
  - 共享变量是否使用 `sync` 包保护
  - `map` 并发访问是否加锁
  - slice 并发修改是否安全

- [ ] **Sync 包使用**
  - `sync.Mutex`/`RWMutex` 是否正确使用
  - `sync.WaitGroup` 是否正确等待
  - `sync.Once` 是否用于单次初始化

## 错误处理审查

- [ ] **错误返回**
  - 错误是否立即返回
  - 是否有错误包装（`fmt.Errorf("%w")`）
  - 错误信息是否有用

- [ ] **Panic 处理**
  - 是否有 `recover()` 机制
  - panic 使用是否恰当（仅用于不可恢复错误）

- [ ] **日志记录**
  - 错误是否有日志记录
  - 日志级别是否正确（INFO/WARN/ERROR/CRITICAL）
  - 日志是否包含足够上下文信息

## 资源管理审查

- [ ] **文件/连接**
  - 文件是否正确关闭
  - 数据库连接是否释放
  - HTTP 响应 Body 是否关闭

- [ ] **内存管理**
  - 是否有内存泄漏风险
  - 大对象是否及时释放
  - slice/map 预分配是否合理

- [ ] **Context 使用**
  - 是否传递 `context.Context`
  - 是否设置超时/截止时间
  - 是否正确处理 `context.Done()`

## 代码质量审查

- [ ] **命名规范**
  - 包名是否简洁（小写，无下划线）
  - 导出标识符是否大驼峰
  - 私有标识符是否小驼峰
  - 变量名是否有意义

- [ ] **代码风格**
  - 是否使用 `gofmt` 格式化
  - 函数长度是否合理（建议<50 行）
  - 注释是否充分（特别是导出标识符）

- [ ] **魔法数字**
  - 常量是否提取为具名常量
  - 配置值是否可配置

- [ ] **性能优化**
  - 字符串拼接是否使用 `strings.Builder`
  - slice 是否预分配容量
  - 循环中是否避免不必要的分配

## 测试覆盖审查

- [ ] **单元测试**
  - 是否有测试文件（`_test.go`）
  - 测试用例是否覆盖核心逻辑
  - 是否有表驱动测试

- [ ] **测试质量**
  - 测试是否独立
  - 测试是否有断言
  - 是否有基准测试（性能关键代码）

## Go 特定问题

- [ ] **Defer 使用**
  - 资源清理是否使用 `defer`
  - `defer` 是否在资源获取后立即使用
  - `defer` 中是否避免使用可变参数

- [ ] **Interface 使用**
  - 是否避免 `interface{}` 滥用
  - 类型断言是否检查 `ok`
  - 是否使用类型开关

- [ ] **Struct 对齐**
  - struct 字段是否按大小对齐
  - 是否有不必要的嵌套结构体

- [ ] **Slice/Map 操作**
  - slice 扩容是否合理
  - map 删除是否使用 `delete()`
  - 是否在遍历中修改 slice/map

## 项目特定检查

- [ ] **日志系统** - 是否统一使用项目日志包
- [ ] **数据库** - 连接管理是否正确
- [ ] **配置文件** - 配置加载是否安全
- [ ] **构建脚本** - 构建脚本是否完整
- [ ] **第三方依赖** - 依赖版本是否锁定

## 常见 Go 陷阱

| 陷阱 | 说明 | 检查点 |
|------|------|--------|
| 循环变量捕获 | 闭包中直接使用循环变量 | `for` 循环中的 goroutine |
| Nil 接口 | 接口值为 nil 但类型非 nil | 接口比较 |
| Slice 内存泄漏 | 大 slice 的子 slice 持有底层数组 | `slice[:]` 操作 |
| Map 迭代顺序 | map 迭代顺序不固定 | `range map` |
| 时间比较 | 时区问题导致时间比较错误 | `time.Time` 比较 |

## 审查报告模板要点

1. **项目概述** - 功能说明、目录结构
2. **优点** - 先肯定做得好的地方
3. **问题分级** - Critical/Major/Minor/Potential
4. **代码统计** - 行数、函数数、测试覆盖率
5. **修复优先级** - 高/中/低
6. **代码对比** - 修复前后的具体代码
