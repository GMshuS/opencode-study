# C++ 代码审查检查清单

## 架构设计审查

### 模块化设计
- [ ] **头文件组织**
  - 头文件是否使用包含保护（`#ifndef HEADER_H_` / `#define` / `#endif`）
  - 是否使用 `extern "C"` 导出 C 风格接口（如需要）
  - 头文件是否只包含必要的依赖（避免循环依赖）
  - 前置声明是否替代了不必要的 `#include`

- [ ] **目录结构**
  - 目录结构是否清晰反映模块划分
  - 公共头文件是否放在 `include/` 目录
  - 实现文件是否放在 `src/` 目录
  - 测试代码是否独立目录（如 `test/` 或 `UnitTest/`）

- [ ] **依赖管理**
  - 依赖关系是否单向（避免循环依赖）
  - 第三方库依赖是否明确
  - 是否使用命名空间避免符号冲突

### 初始化机制
- [ ] **全局/静态变量**
  - 全局变量初始化是否线程安全
  - 是否避免使用全局对象（静态初始化顺序问题）
  - 单例模式是否使用 `Meyers Singleton`（C++11 局部静态变量）

- [ ] **资源清理**
  - 是否有对应的反初始化函数
  - 资源清理是否完整（内存、连接、句柄）
  - 是否使用 RAII 管理资源

### 接口设计
- [ ] **API 设计**
  - 导出接口是否简洁、稳定
  - 是否使用 `__declspec(dllexport/dllimport)` 控制符号导出
  - 参数是否使用 `const` 修饰（只读参数）
  - 输出参数是否有清晰的文档说明

## 内存管理审查

### 动态内存
- [ ] **内存分配**
  - 是否使用智能指针（`std::unique_ptr`/`std::shared_ptr`）
  - `new/delete` 是否配对使用
  - 数组是否使用 `new[]/delete[]` 配对
  - 是否避免裸指针所有权转移

- [ ] **内存泄漏**
  - 是否有内存泄漏检测机制
  - 异常路径下资源是否释放
  - 循环引用是否避免（`std::weak_ptr`）

### 缓冲区安全
- [ ] **缓冲区溢出**
  - 字符串拷贝是否使用安全函数（`snprintf`/`strncpy`/`strcpy_s`）
  - 数组访问是否边界检查
  - 缓冲区大小是否显式传递
  - 是否使用 `std::string`/`std::vector` 替代裸数组

## 并发安全审查

### 线程同步
- [ ] **锁机制**
  - 共享数据是否加锁保护
  - 是否使用读写锁（`std::shared_mutex`）区分读写操作
  - 是否避免死锁（锁顺序一致）
  - 锁粒度是否合理（避免过大或过小）

- [ ] **原子操作**
  - 计数器是否使用原子操作（`std::atomic`）
  - 标志位是否使用 `std::atomic<bool>`
  - 是否避免不必要的锁（使用无锁数据结构）

### 线程安全模式
- [ ] **引用计数**
  - 引用计数增减是否原子操作
  - `AddRef()`/`Release()` 是否配对调用
  - 零引用时对象销毁是否安全

- [ ] **线程局部存储**
  - 是否正确使用 `thread_local`
  - 线程局部数据清理是否处理

### 跨平台并发
- [ ] **平台差异**
  - Windows (`InterlockedXXX`) 与 Linux (`__sync_xxx`/`__atomic_xxx`) 原子操作是否统一封装
  - 线程 API 是否封装（`std::thread` 优先）
  - 条件变量是否正确处理虚假唤醒

## 错误处理审查

### 异常处理
- [ ] **异常安全**
  - 是否有异常捕获机制（特别是边界处）
  - 析构函数是否 `noexcept`（不抛出异常）
  - 异常是否及时捕获并记录
  - 是否避免在异常路径泄漏资源

- [ ] **错误码**
  - 返回值是否有明确的错误码定义
  - 错误码是否统一（如 0 成功，非 0 失败）
  - 错误信息是否传递到调用方
  - 是否有全局异常处理器（如 `CGloabalExceptionProc`）

### 断言
- [ ] **运行时检查**
  - 关键假设是否使用 `assert()` 检查
  - 输入参数是否验证（空指针、边界）
  - `assert()` 是否仅用于调试（非错误处理）

## 资源管理审查

### 文件/连接
- [ ] **文件操作**
  - 文件打开后是否正确关闭
  - 文件路径是否跨平台兼容
  - 文件读写错误是否检查

- [ ] **网络连接**
  - Socket 连接是否正确关闭
  - 连接超时是否处理
  - 重连机制是否完善

### 句柄管理
- [ ] **系统句柄**
  - 数据库连接是否释放
  - 线程句柄是否关闭
  - 互斥量/信号量是否销毁

## 代码质量审查

### 命名规范
- [ ] **标识符命名**
  - 类名是否使用大驼峰（`ClassName`）
  - 函数名是否使用大驼峰（`FunctionName`）
  - 成员变量是否有前缀（`m_` 或 `m_`）
  - 常量是否全大写（`CONSTANT_NAME`）
  - 私有成员是否有访问约定

- [ ] **文件命名**
  - 头文件是否 `.h`/`.hpp`
  - 实现文件是否 `.cpp`
  - 文件名是否有意义

### 代码风格
- [ ] **格式化**
  - 缩进是否一致（空格或 Tab）
  - 大括号风格是否统一
  - 行宽是否合理（建议<120 字符）
  - 空行是否合理使用

- [ ] **注释**
  - 头文件是否有文件说明（`@file`）
  - 函数是否有参数/返回值说明
  - 复杂算法是否有注释
  - 注释是否使用中文（项目约定）
  - 修改历史是否记录（`@version`）

- [ ] **函数设计**
  - 函数长度是否合理（建议<50 行）
  - 参数个数是否合理（建议≤5 个）
  - 是否提取重复代码为独立函数

### 魔法数字
- [ ] **常量提取**
  - 魔法数字是否提取为具名常量
  - 配置值是否可配置
  - 是否使用 `enum`/`constexpr` 定义常量

## 性能优化审查

### 对象生命周期
- [ ] **拷贝控制**
  - 是否实现拷贝构造函数
  - 是否实现拷贝赋值运算符
  - 是否实现移动语义（C++11）
  - 是否使用 `delete` 禁止拷贝（单例等）

- [ ] **虚函数**
  - 基类析构函数是否 `virtual`
  - 虚函数表指针是否影响内存布局
  - 是否避免虚函数构造函数中调用

### 容器使用
- [ ] **STL 容器**
  - `std::vector` 是否预分配容量（`reserve()`）
  - `std::map` vs `std::unordered_map` 选择是否合理
  - 迭代器失效位是否处理
  - 遍历时是否避免修改容器

### 字符串处理
- [ ] **字符串优化**
  - 字符串拼接是否使用 `std::ostringstream`
  - 是否避免临时字符串创建
  - C 字符串与 `std::string` 转换是否最小化

## 测试覆盖审查

### 单元测试
- [ ] **测试框架**
  - 是否使用测试框架（如 Google Test）
  - 测试文件是否独立（`*Test.cpp`）
  - 测试用例是否覆盖核心逻辑

- [ ] **测试质量**
  - 测试是否独立（无依赖）
  - 测试是否有断言
  - 边界条件是否测试
  - 异常路径是否测试

### 集成测试
- [ ] **端到端测试**
  - 是否有集成测试
  - 是否测试跨模块交互
  - 是否测试并发场景

## C++ 特定问题

### 头文件包含
- [ ] **包含顺序**
  - 自己的头文件是否优先
  - 系统头文件最后
  - 是否使用相对路径
  - 是否避免冗余包含

### 类型安全
- [ ] **类型转换**
  - 是否避免 C 风格强制转换
  - 是否使用 `static_cast`/`dynamic_cast`
  - `reinterpret_cast` 是否仅用于底层代码

- [ ] **整数溢出**
  - 整数运算是否检查溢出
  - 有符号/无符号混合是否避免
  - `size_t` vs `int` 选择是否合理

### 未定义行为
- [ ] **常见陷阱**
  - 是否避免悬空指针
  - 是否避免使用未初始化变量
  - 是否避免序列点问题
  - 是否避免 strict aliasing 违规

### C++11/14/17 特性
- [ ] **现代 C++**
  - 是否使用 `auto` 简化类型
  - 是否使用范围 `for` 循环
  - 是否使用 `nullptr` 替代 `NULL`
  - 是否使用 `override`/`final`
  - 是否使用 `std::unique_ptr`/`std::shared_ptr`

## 跨平台审查

### 平台兼容
- [ ] **条件编译**
  - 是否使用 `#ifdef _WIN32` 区分平台
  - Linux 代码是否测试
  - 字节序是否处理（大端/小端）

- [ ] **数据类型**
  - 是否使用固定宽度类型（`int32_t` 等）
  - `long` 与 `long long` 选择是否合理
  - 指针与整数转换是否安全

### 构建系统
- [ ] **编译配置**
  - 构建脚本是否完整
  - Debug/Release 配置是否正确
  - 警告级别是否合理（`-Wall -Wextra`）
  - C++ 标准是否指定（`-std=c++11`）

## 常见 C++ 陷阱

| 陷阱 | 说明 | 检查点 |
|------|------|--------|
| 切片引用 | 容器迭代器/引用在容器修改后失效 | `vector`/`string` 扩容 |
|  dangling 引用 | 返回局部变量引用 | 函数返回值 |
| 虚函数析构 | 基类析构非虚导致泄漏 | 多态基类 |
| 拷贝省略 | RVO/NRVO 依赖 | 返回值优化 |
| 静态初始化 | 跨文件静态变量初始化顺序 | 全局对象 |
| 整数提升 | 有符号/无符号混合运算 | 类型转换 |
| 序列点 | 表达式求值顺序未定义 | `i = i++` |
| 严格别名 | 不同类型指针别名违规 | `reinterpret_cast` |
| 移动语义 | 移动后对象状态未定义 | `std::move()` 后使用 |

## 审查报告模板要点

1. **项目概述** - 功能说明、目录结构
2. **优点** - 先肯定做得好的地方
3. **问题分级** - Critical/Major/Minor/Potential
4. **代码统计** - 行数、函数数、测试覆盖率
5. **修复优先级** - 高/中/低
6. **代码对比** - 修复前后的具体代码
