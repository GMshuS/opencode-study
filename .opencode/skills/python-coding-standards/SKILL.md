# 核心规范摘要
1. 遵循 PEP 8 编码规范（缩进、空格、换行）
2. 使用有意义的命名（蛇形命名法、类名大驼峰）
3. 使用类型注解（函数签名、变量声明）
4. 错误处理优先（使用 try/except、捕获具体异常）
5. 遵循文档字符串规范（模块、类、函数注释）
6. 避免可变默认参数
7. 使用列表/字典推导式（简单情况）
8. 使用 black、flake8、mypy 进行代码检查
9. 遵循项目结构规范（src/tests 分层）
