# Git 自动提交助手

请按以下步骤执行：

1. **检查状态**：运行 `git status --short` 查看变更，如果传入了`指定目录或文件`参数，则只检查该范围内的变更
2. **分析变更**：运行 `git diff` 分析修改内容
3. **分类处理**：
   - 新增文件 → `git add <文件>`
   - 修改文件 → `git add <文件>`
   - 删除文件 → `git rm <文件>`
4. **生成提交信息**：包含`问题来源`、`问题原因`、`修改说明`、`测试建议`四块内容，格式如下：

```text
问题来源：【问题的来源与描述】
问题原因：【问题产生的原因】
修改说明：【修改的方法以及修改点】
测试建议：【对修改给出测试建议，涵盖修改点】
```

注意：
- `问题来源`给出两个选项`自动生成`跟`自定义`，如果用户选择`自定义`则用户输入`问题来源`的描述，否则 AI 自动生成描述信息
- **提交信息**必须要完整展示给用户

5. **执行提交**：`git commit -m "**生成提交信息**"`
6. **推送**（可选）：`git push`

执行过程严格遵守以下规则：
- 每次**执行提交**跟**推送**前必须询问用户确认
- 避免使用 `git add .`，明确指定文件
- `指定目录或文件`参数用于设置提交的目标范围，如果未指定则处理所有变更
