# Git 自动提交助手

## 参数说明

本命令支持通过两种方式指定提交范围：

### 方式 A：由 dev-master 传递上下文（coding-dev 流程）
```
- 本次修改的所有代码文件路径: [文件列表，每行一个]
- 排除目录: coding-dev/
```
此时**仅提交指定文件列表中的变更**，忽略其他未指定文件的修改。

### 方式 B：手动指定目录或文件
传入 `指定目录或文件` 参数，只处理该范围内的变更。
如果两者均未指定，则处理所有变更（`git status --short` 检测到的全部文件）。

## 执行步骤

### 1. 确定提交范围
- 如果已传入"本次修改的所有代码文件路径"列表 → 以此为准，逐一检查文件是否存在变更
- 否则如果传入了`指定目录或文件`参数 → 运行 `git status --short <参数>` 限定范围
- 否则 → 运行 `git status --short` 查看全部变更

### 2. 过滤排除目录
如果指定了排除目录（如 `coding-dev/`）：
- 从提交列表中移除匹配该目录的所有文件
- 输出排除的文件名供用户知晓

### 3. 分析变更
对最终确定要提交的文件列表：
- 逐一运行 `git diff <文件>` 分析修改内容
- 新增文件 → `git add <文件>`
- 修改文件 → `git add <文件>`
- 删除文件 → `git rm <文件>`

### 4. 生成提交信息
包含`问题来源`、`问题原因`、`修改说明`、`测试建议`四块内容：

```text
问题来源：【问题的来源与描述】
问题原因：【问题产生的原因】
修改说明：【修改的方法以及修改点】
测试建议：【对修改给出测试建议，涵盖修改点】
```

注意：
- `问题来源`给出两个选项`自动生成`跟`自定义`，如果用户选择`自定义`则用户输入描述，否则 AI 自动生成
- **提交信息**必须要完整展示给用户确认

### 5. 执行提交
`git commit -m "**提交信息**"`

### 6. 推送（可选）
`git push`

## 规则

- **执行提交**和**推送**前必须询问用户确认
- **避免使用 `git add .`**，必须逐一明确指定文件
- 排除目录优先于提交范围（先确定范围，再排除）
- 如果过滤后无文件可提交，告知用户并终止流程
