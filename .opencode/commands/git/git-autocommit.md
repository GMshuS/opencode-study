# Git 自动提交助手

请按以下步骤执行：

1. **检查状态**：运行 `git status --short` 查看变更
2. **分析变更**：运行 `git diff` 分析修改内容
3. **分类处理**：
   - 新增文件 → `git add <文件>`
   - 修改文件 → `git add <文件>`
   - 删除文件 → `git rm <文件>`
4. **生成提交信息**：包含`问题来源`、`问题原因`、`修改说明`、`测试建议`四块内容，格式如下：

```text
问题来源：【问题的来源与描述】
问题原因：【问题产生的原因】
修改说明：【修改的方法以及修改点】
测试建议：【对修改给出测试建议，涵盖修改点】
```

注意：`问题来源`需要询问用户，用户可以修改

5. **执行提交**：`git commit -m "问题来源：..." -m "问题原因：..." -m "修改说明：..." -m "测试建议：..."`
6. **推送**（可选）：`git push`

注意：
- 每次提交前询问用户确认
- 避免使用 `git add .`，明确指定文件
