# Git 自动提交助手

严格遵守以下流程执行提交操作。

## 步骤1：检查状态

运行 `git status --short` 查看变更。

如果传入了**指定目录或文件**参数，则只检查该范围内的变更。

## 步骤2：分析变更

运行 `git diff`（指定了范围时加文件路径限定）分析修改内容。

## 步骤3：分类处理

逐个分析变更文件，按类型执行：
- 新增文件 → `git add <文件>`
- 修改文件 → `git add <文件>`
- 删除文件 → `git rm <文件>`

**禁止使用 `git add .`**，必须明确指定每个文件。

## 步骤4：生成提交信息

生成包含以下四块内容的提交信息：

```text
问题来源：【问题的来源与描述】
问题原因：【问题产生的原因】
修改说明：【修改的方法以及修改点】
测试建议：【对修改给出测试建议，涵盖修改点】
```

**问题来源规则**：
- 先询问用户选择「自动生成」或「自定义」
- **自动生成**：AI 根据变更内容自动分析并生成描述
- **自定义**：用户输入问题来源的描述

**提交信息**必须完整展示给用户确认。

## 步骤5：执行提交

`git commit -m "<提交信息>"`

**执行前必须询问用户确认。**

## 步骤6：推送（可选）

`git push`

**执行前必须询问用户确认。**
