## Execution

## Task Planning Guidelines

When creating the task plan, follow these principles to MINIMIZE dependencies between tasks:

1. **Parallel-First Thinking**: Group tasks that can be executed in parallel
   - Tasks with no data dependency should have NO dependencies
   - Only add dependencies when one task's output is truly required by another

2. **Dependency Criteria**: A task should depend on another ONLY if:
   - Its input is another task's output
   - It requires data/result generated by the dependent task
   - NOT just because "they're related" or "should be done together"

3. **Independent by Default**: Default to NO dependencies
   - If task B only needs the same inputs as task A, they are INDEPENDENT
   - Shared resources (like database schema) are NOT task dependencies

4. **Task Granularity**: Prefer more granular independent tasks over fewer dependent ones
   - This enables maximum parallelization across AI conversations
   - Each task should be self-contained when possible

## Example of Good vs Bad Dependencies

BAD (too many dependencies):

- Task 1: Create API routes → depends on []
- Task 2: Create database schema → depends on [Task 1] ← WRONG
- Task 3: Write API tests → depends on [Task 1, Task 2] ← UNNECESSARY

GOOD (minimal dependencies):

- Task 1: Create database schema → depends on []
- Task 2: Create API routes → depends on [] ← INDEPENDENT
- Task 3: Write API tests → depends on [Task 2] ← ONLY if truly needed

Analyze the following requirements and create a structured task plan:

**Project Name**: $PROJECT_NAME

**Requirements**: $REQUIREMENTS

Create a `tasks/$PROJECT_NAME/task_plan.md` file with the following structure:

```markdown
# Task Plan: $PROJECT_NAME

## Project Overview

[Brief description of the project]

## Tasks

### Task 1: [Task Name]

- **ID**: task-1
- **Description**: [What this task does]
- **Dependencies**: []
- **Input**: [What this task needs as input]
- **Output**: [What this task produces]

### Task 2: [Task Name]

- **ID**: task-2
- **Description**: [What this task does]
- **Dependencies**: [task-1]
- **Input**: [What this task needs as input]
- **Output**: [What this task produces]

[Continue for all tasks...]
```

The task name (e.g., "project-a", "feature-b") will be used to create an isolated directory. First check the project context to understand what needs to be built.

Output the task plan and explain the reasoning behind each task and its dependencies.

## Wait for User Confirmation

After generating the task plan, OUTPUT a clear instruction like:

```
────────────────────────────────────────
📋 Task Plan Generated: $PROJECT_NAME

Total Tasks: [number]
Estimated Parallel Groups: [number]

To execute the plan, choose one:

1. Sequential execution (one task at a time):
   /bigtask-execute $PROJECT_NAME

2. Execute specific task:
   /bigtask-execute $PROJECT_NAME/task-1

3. Modify the plan:
   [Provide your feedback]
────────────────────────────────────────
```

DO NOT automatically execute any tasks. Only output instructions, do not wait for response.
