设计一套可以在opencode等工具中使用的自定义命令系统

#目的：
由于AI单次对话上下文有限，关键在于如何在对话之间传递状态。


#核心思路：
状态持久化 + 任务编排以下是几种实用方案：


#方案：

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  对话 1      │────→│  状态文件    │←────│  对话 2      │
│ (任务1执行)  │     │ (JSON/YAML) │     │ (任务2执行)  │
└─────────────┘     └─────────────┘     └─────────────┘
         ↑                                    ↓
         └────────────┐    ┌─────────────────┘
                      ↓    ↓
              ┌───────────────┐
              │   主控程序     │
              │ (拆分&调度任务) │
              └───────────────┘
```

#实现方式：

1. 任务拆分阶段（对话1）
   - 上传完整需求，让AI生成任务清单并保存为结构化文件
   - 输出：`task_plan.md` 包含任务ID、依赖关系、输入输出定义

2. 执行阶段（多个独立对话）
   - 每个对话只处理一个子任务
   - 读取前置任务的输出文件作为输入
   - 完成后写入新的状态文件

3. 聚合阶段（最终对话）
   - 读取所有子任务的输出，整合为最终结果
